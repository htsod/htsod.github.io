Motivation#

Linux is such a amazing tool for learning computer operation. It provides transparency and customizability to its users. Unlike Windows and Mac that sometimes act as a barrier to communicate with the hardwares, with Linux, for the very first time I feel like I am the one that control the machine.

Despite the pros, getting our hands dirty on Linux operating system is essential for a better user experience. To this end, the book “THe Linux Command Line” provides a helpful introduction and user hand book. The core information from this book is that Linux built itself from a file hierachy and any file operation could be done by a built-in programming language called shell.

Though I have read through the book, whenever I comes across some practical problem in Linux I have to filp over the book for review and solution, which is not very convenient for fast answer. If I look the answer through internet, the answer would lack the conceptual understanding that connects other concept of the Linux system. Hence, a compromising solution is to have a comprehensive and yet connected summary note from the book, and why this blog comes about.

Wildcards

  • *

Matches any characters

  • ?

Matches any single character

  • [characters]

Matches any character that is a member of the set characters

  • [!characters]

Matches any character that is not a memer of the set characters

  • [[:class:]]

Matces any character that is a member of the specified class

For example, [:alnum:] matches any alphanumeric character. [:alpha:] matches any alphabetic character. [:digit:] matches any numeral, [:lower:] matches any lowercase letter, [:upper:] matches any uppercase letter.
